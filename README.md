# KnowTune
LLM-guided tuning is effective due to its leverage of domain knowledge. Unfortunately, existing approaches suffer from a fundamental limitation: they fail to account for knowledge criticality, which varies with workload runtime characteristics. They fail to account for knowledge criticality, which refers to the importance of specific knowledge elements (such as parameter tuning experience) on the overall performance improvements and varies with workload runtime characteristics. For instance, when tuning MySQL for I/O-intensive workloads, current methods tune memory and I/O parameters based on cross-scenario averages. However, memory-related parameters tend to have a smaller performance impact, while more I/O-related parameters are critical due to the I/O bottleneck.

To bridge the gap between domain knowledge and workload characteristics, we propose knowledge attention to measure the knowledge criticality. The key challenges are calculating attention and clarifying LLM outputs derived from it. KnowTune first monitors runtime metrics to identify performance bottlenecks, enabling a two-stage attention reasoning strategy. We then extend LLM outputs to include recommended parameters, value ranges, starting points, and their weights. Using these outputs, KnowTune employs Bayesian optimization to find the final configurations.
